# GitHub access

This project lives on GitHub, there is a mcp-tool for accessing additional github areas, such as issues.
This tool is named github-lassevk.

# Rules for code and development

* All variables, typenames, methodnames, comments, etc. must be written in english
* Same goes for all text messages that can or will be viewed by humans
* All code blocks must use `{ }` syntax, even if they're a single statement in the if-then branch of a if-statement
* Solve issues as presented, do not solve future problems. If a future problem is likely, discuss it with me before dismissing it.
* All compiler warnings must be handled with the same priority as compiler errors.
* When committing code to git, split changes into logical coherent commits, if possible
* Do not blindly assume that I covered or remembered everything in my issues. Challenge my assumptions and discuss when needed.
